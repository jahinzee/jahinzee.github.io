The **B**anner **A**lgebraic[^1] and **N**ormalised **Notation** (*BANNotation*) is a notation system that I devised for communicating banner designs in *Minecraft* (Mojang, 2011). This page will describe this system, and provide some examples of its use.

## Syntax

A banner notation starts with a base, and then any number of patterns, in order of their application.

### Base

> *For a more formal syntax definition, see [Appendix: Syntax in EBNF](#syntax-in-ebnf).*

A base banner is simply defined as <Latex>B</Latex>, where <Latex>B</Latex> is the base colour.

### Simple Patterns

A colour-pattern combination is defined as <Latex>C</Latex>-<Latex>P</Latex>, where <Latex>C</Latex> is a colour, and <Latex>P</Latex> is a pattern. For example, a Blue Chevron would be **Bl-Tb** *(blue triangle bottom)*.

### Combined Gradient Base

For simple gradient bases, instead of notating a base <Latex>C_A</Latex> and a gradient bottom <Latex>C_B</Latex> (or the other way with a gradient top), you can use the shorthand **<Latex>C_A</Latex>+<Latex>C_B</Latex>-Gx.**, where **Gx** is the combined gradient pseudo-pattern, <Latex>C_A</Latex> is the top colour, and <Latex>C_B</Latex> is the bottom colour.

For example, a gradient that goes from blue at the top and red at the bottom would can be either:

- **Rd. Bl-Gt.**, or
- **Bl. Rd-Gb.**

But with the combined gradient syntax, it would be:

- **Bl+Rd-Gx.**

Note that this only works if the gradient is applied immediately after the base. If there are any patterns that are applied in between, the normal gradient top/bottom should be used instead.

## Appendices

### Syntax in EBNF 

```ebnf
colour = "Wh"
       | "Lg"
       | "Gr"
       | "Bk"
       | "Br"
       | "Rd"
       | "Or"
       | "Ye"
       | "Li"
       | "Gn"
       | "Cy"
       | "Lb"
       | "Bl"
       | "Pu"
       | "Ma"
       | "Pi"

pattern = "Sb"
        | "St"
        | "Sl"
        | "Sr"
        | "Scv"
        | "Sch"
        | "Sdw"
        | "Suw"
        | "Sbar"
        | "Sx"
        | "Scr"
        | "Wtl"
        | "Wtr"
        | "Wbl"
        | "Wbr"
        | "Bl"
        | "Br"
        | "Bt"
        | "Bb"
        | "Bbl"
        | "Bbr"
        | "Btl"
        | "Btr"
        | "Tb"
        | "Tt"
        | "Tmb"
        | "Tmt"
        | "Cc"
        | "Cd"
        | "F"
        | "Fw"
        | "Cb"
        | "Gt"
        | "Gb"
        | "Cr"
        | "Cs"
        | "Cf"
        | "Cn"
        | "Cg"
        | "Cp"

solid = colour, "."
gradient = colour, "+", colour, "-Gx."
base = solid | gradient

pattern_set = pattern, { "+", pattern }
layer = colour, "-", pattern_set, "."

banner = base, { " ", layer }
```

[^1]: "algebraic", as in [Algebraic Notation](https://en.wikipedia.org/wiki/Algebraic_notation_(chess)) in chess.
[^2]: This is a crucial point, as there are some naming conflicts between Bedrock and Java Edition, mainly in the Per Bends.
